Adopt OpenGate without guessing.

This docs site follows the published package versions. The root package stays compatible, the adapter packages are explicit, and the starter templates show the intended install paths.

Start with the install guide, then move to the template that matches your app: website, API, or partner/server-to-server.

Install matrix

Need Install path Starter

Starter templates

Website

Browser login, HttpOnly JWT cookie, free-tier to paid-tier upgrade, and a hidden `/api` route.

API

A lean JWT-protected JSON API for existing services that want a small integration surface.

Partner

Server-to-server API-key flow with explicit client credentials and a narrow starter shape.

What changed in Phase 7

  • Workspace packaging for compatibility-first distribution.
  • Explicit adapter packages for Fastify and Express.
  • Versioned docs and Changesets-based release flow.
  • Starter templates that match the common adoption paths.
