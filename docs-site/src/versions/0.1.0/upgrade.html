Versioning model

OpenGate uses Changesets for coordinated package versioning and changelog generation. The root package stays backward compatible while adapter packages and starter docs can move in lockstep.

Upgrade checklist

  • Keep the root opengate import if you are on the compatibility path.
  • Move to explicit adapter packages when you want framework-specific installs.
  • Update starter usage from --route to --template when you want the flagship path.

Release notes

Release notes are generated from changesets and shipped with each published package set. This keeps package upgrades and docs in sync.
